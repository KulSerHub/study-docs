# study-docs — изучить документацию по ссылке

Задача: загрузить документацию (по ссылке или найденную по теме), извлечь чистый
текст статьи с сохранением ссылок и картинок, прочитать дословно — и дальше отвечать
строго по этому тексту, без выдумок.

## Чем разбирать страницы

Скиллу гарантированно доступен только **Node.js**. На всех шагах — разведка,
поиск ссылок, разбор HTML — опирайся лишь на то, что точно есть:

- **Поиск и чтение** (ссылки, структура раздела, списки страниц в скачанном
  HTML): встроенные инструменты `Grep` и `Read` по файлам в `$DIR`.
- **Скачивание**: `curl`.
- **Извлечение текста**: только `node clean-html.cjs` (шаг 4).
- Нужен разовый вспомогательный скрипт — пиши его на **Node** и запускай `node …`.
- **Не используй WebFetch** для текста доков — он пересказывает страницу через
  модель-суммаризатор и теряет/искажает детали. Точный текст даёт только
  скачанный файл.

**Не используй Python** и другие незаявленные зависимости — всё делаем через
Node.js и встроенные инструменты.

## Подготовка (один раз на проект)

Чистильщику нужны npm-зависимости. Если рядом со скиллом нет папки `node_modules` —
установи их:
```
npm install --prefix "<папка-этого-скилла>"
```
(зависимости: `@mozilla/readability`, `jsdom`, `turndown`, `turndown-plugin-gfm`).

## Шаги

1. **Определи источник.**
   - Дана ссылка → используй её.
   - Дана только тема → найди официальную документацию (WebSearch), покажи найденный
     URL и кратко подтверди, что берём именно его.

2. **Заведи папку для кэша внутри скилла** (файлы сохраняются — пользователь сам
   решает, когда удалять):
   ```
   DIR="<папка-этого-скилла>/docs-cache/<короткий-slug>"
   mkdir -p "$DIR"
   ```
   Переменные шелла **не сохраняются между вызовами Bash** — задавай `DIR`
   заново в каждой команде или подставляй полный путь.

3. **Скачай страницы** через `curl -fsL <url> -o "$DIR/<имя>.html"`
   (`-f` — при HTTP-ошибке файл не создаётся и команда падает, ошибку видно сразу).
   - **Качай только в `$DIR`. Никогда не используй `/tmp` и другие папки** — даже
     первую/разведочную страницу для анализа структуры скачивай сразу в `$DIR`.
   - Имена файлов делай **уникальными**: последние сегменты URL у разных страниц
     могут совпадать (`/guide/intro`, `/api/intro`) — одноимённый файл молча
     перезапишет предыдущий. При совпадении добавляй префикс родительского
     раздела: `guide-intro.html`, `api-intro.html`.
   - Просили одну страницу → скачай только её, раздел не собирай.
   - Нужен весь раздел → раздел это **именованная группа в меню/сайдбаре сайта**,
     а не общий префикс URL. Скачай страницу раздела в `$DIR` и найди в её HTML
     (через Grep/Read) блок навигации с заголовком, который назвал пользователь;
     собери все ссылки этого блока до следующего заголовка группы.
   - **Сверь список с пользователем.** Покажи найденные страницы с их названиями
     как в меню и дождись подтверждения — пользователь видит сайдбар на сайте
     и заметит пропуски. Если в блоке есть подзаголовок, под которым нет ни одной
     ссылки — это свёрнутая подгруппа, чьи пункты подгружаются скриптом; честно
     скажи об этом и попроси пользователя перечислить недостающие пункты или
     дать ссылку на любую страницу подгруппы.
   - **Проверь llms.txt** (для раздела — после подтверждения списка, для одной
     страницы — сразу) — необязательный машинный индекс сайта со ссылками на
     готовые Markdown-версии страниц. Ищи его у корня документации, затем у
     корня сайта (`<база-доков>/llms.txt`, `<домен>/llms.txt`). Если файл
     есть — сопоставь нужные страницы с его записями по пути URL и для
     совпавших качай их `.md`-ссылку:
     `curl -fsL --create-dirs -o "$DIR/<имя>/<имя>.md" <url>`. Это первоисточник,
     очистка (шаг 4) ему не нужна. Если в скачанном файле HTML-теги вместо
     markdown — отбрось его и качай страницу как HTML. Нет llms.txt или нет
     записи — качай HTML как обычно, это не ошибка.
     Для страницы с готовым `.md` HTML не нужен: если её `.html` уже скачан
     (например, разведочный для поиска раздела) — **удали его до шага 4**,
     иначе скрипт перезапишет первоисточник своей версией.
   - **Проверь скачанное**: пустой или крошечный файл — признак ошибки
     (404, редирект, защита от ботов). Перекачай или сообщи пользователю.
   - Не обходи весь сайт — только запрошенный раздел.

4. **Извлеки текст HTML → Markdown** (сохраняет `[текст](ссылка)` и `![alt](картинка)`):
   ```
   node "<папка-этого-скилла>/clean-html.cjs" "$DIR" <домен>
   ```
   Передавай домен (напр. `https://code.visualstudio.com`), чтобы относительные ссылки
   и картинки стали абсолютными — тогда потом понятно, «куда копать дальше».
   Скрипт раскладывает каждую страницу в свою подпапку: `$DIR/<имя>/<имя>.html` +
   `$DIR/<имя>/<имя>.md`. Он обрабатывает только `.html` — готовые `.md` из
   llms.txt не трогает. Если итоговый `.md` пустой или почти пустой — тело
   страницы рендерится скриптом в браузере и в HTML его нет; честно сообщи об
   этом пользователю и не отвечай по этой странице из памяти.

5. **Прочитай все `*.md`** в `$DIR` (рекурсивно) инструментом Read — это даёт
   дословный текст (в отличие от пересказа WebFetch). Если страниц очень много
   (десятки), предупреди пользователя, что всё сразу может не поместиться в
   контекст, и предложи разбить чтение на части.

6. **Отчитайся коротко:** что загружено (список страниц), где лежит (`$DIR` в проекте),
   и что готов отвечать по этому тексту.

## Правила ответов после загрузки

- Отвечай **только из загруженного текста**. Чего там нет — так и скажи; если
  отвечаешь из общих знаний, **явно это помечай**.
- Для точных имён настроек/команд/кнопок опирайся на дословный текст, а не на память.
